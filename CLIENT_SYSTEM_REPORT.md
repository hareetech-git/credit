# Udhaar Capital
## Client Report: What Was Built and How It Works

---

## 1. Project Summary

This is a complete loan and customer management platform with four separate user panels:

1. Admin Panel
2. Staff Panel
3. DSA Partner Panel
4. Customer Panel

The system helps your business:

1. Collect loan applications online
2. Review and process applications quickly
3. Track customer conversations and follow-ups
4. Manage DSA partner onboarding and leads
5. Control services, FAQs, team members, and website contact details

---

## 2. Who Uses What

## 2.1 Admin

Admin controls the full business process.

Admin can:

1. Manage staff, customers, and DSA partners
2. Assign loan applications to staff
3. Approve or reject loan and DSA requests
4. Track and manage all customer enquiries
5. Manage website services and content
6. Update FAQs, team profile section, and contact details

## 2.2 Staff

Staff handles day-to-day operations based on the access given by admin.

Staff can:

1. Handle assigned loan applications
2. Verify customer documents
3. Update application status
4. Manage assigned enquiries and customer conversations
5. Manage customers (if allowed)
6. Manage FAQs (if allowed)

## 2.3 DSA Partner

DSA partner focuses on lead generation.

DSA can:

1. Submit new loan leads
2. Track lead status
3. Manage own profile details
4. View dashboard performance

## 2.4 Customer

Customer can:

1. Register and login
2. Complete profile and basic details
3. Submit loan applications
4. Upload required documents
5. Track application status
6. Raise enquiries and chat with support team
7. Apply to become a DSA partner

## 7. Full Business Flow (Start to End)

## 7.1 Customer Journey

1. Customer registers or logs in
2. Customer fills profile details
3. Customer applies for loan
4. Customer uploads required documents
5. Application enters review queue
6. Customer tracks status from own panel
7. Customer can raise enquiry anytime

## 7.2 Internal Loan Processing Journey

1. Admin sees new applications
2. Admin assigns application to staff (or staff takes assigned work)
3. Staff reviews profile and documents
4. Staff verifies or rejects documents if needed
5. Staff/admin updates application decision
6. Customer receives update communication

## 7.3 Enquiry Management Journey

1. Enquiry can come from website form or customer panel
2. Admin assigns enquiry to a staff owner
3. Team and customer can communicate in one conversation thread
4. Team can add internal notes for coordination
5. Enquiry moves through stages until converted or closed

## 7.4 DSA Partner Journey

1. User submits "Become DSA" request
2. Admin reviews request details
3. Admin approves or rejects
4. On approval, DSA account is activated
5. DSA logs in and starts submitting leads
6. DSA tracks all own leads in panel

## 7.5 Content Management Journey

1. Admin builds service structure (category -> subcategory -> service)
2. Admin defines service-related details and required documents
3. Customer sees these services on website and applies accordingly

## 8. Current Communication / Email Flow (Already Working)

The system already sends important communications for:

1. New loan application received (to admin team)
2. Loan assigned to staff (to assigned staff)
3. Loan approved or rejected (to customer)
4. All documents verified (to customer)
5. New DSA request submitted (to admin team)
6. DSA request approved or rejected (to applicant)
7. Enquiry email sent manually from admin/staff panel
8. Career application sent to HR email

---

## 9. Where More Notifications Can Be Added (Business Recommendation)

To improve customer experience and response speed, recommended additions are:

1. Send customer confirmation immediately after application submission
2. Send customer alert when enquiry gets assigned to a staff member
3. Send internal alert when any new enquiry comes in
4. Send customer alert when enquiry is marked converted/closed
5. Send staff alert when career application is assigned
6. Send welcome details when new staff account is created
7. Send welcome details when admin creates DSA account directly
8. Send security alert when account password is changed
